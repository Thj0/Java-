# Escolhendo IDE

Para aumentar nossa produtividade, nós podemos escolher algumas ferramentas denominadas de **IDE - Integrated Development Environment** ou **Ambiente de Desenvolvimento Integrado.** É um software que dispõe de recursos que auxiliam muito o nosso desenvolvimento como: 

- Aceleração de nossa escrita com **Auto-complete ou IntelliSense.**

- Formatação de palavras e blocos de códigos.

- Análise de erro de sintaxe.

- Compilação de programa.

- Depuração (acompanhamento) de execução do programa

  Identação - Espaçamento.

Cada IDE dispõe de recursos comuns ou plugins específicos para te auxiliar na agilidade de desenvolvimento.



***Cada IDE possui suas particularidades.***

1. Visual Studio Code.
2. Apache NetBeans IDE.
3. Eclipse.
4. IntelliJ IDE.



## Eclipse 

O [Eclipse IDE](https://eclipseide.org/) é uma alternativa muito comum de encontrar ambientes de desenvolvimento, principalmente pelo aspecto de uma interface simples, plugins para recursos adicionais, atalhos para operações de menu e de escrita, e para quem trabalha com Java Swing. Você pode adicionar o plugin [WindowBuilder](https://eclipse.dev/windowbuilder/) que dispõe de uma interface gráfica para criar as telas gerando o código de layout para o desenvolvedor. 



## IntelliJ

O [IntelliJ](https://www.jetbrains.com/pt-br/idea/) é uma plataforma de código aberto para criação de IDE's e ferramentas para desenvolvedores, mas nem todos os recursos estão disponíveis de forma gratuita.

Mesmo com a proposta de ser inteligente, talvez em primeiro contato é difícil já desfrutar de todos os recursos, fazendo com que você opte por mudar ou voltar para uma IDE como o Eclipse. Claro que não podemos deixar de destacar que muitas coisas a gente não precisa, como: Salvar arquivo, encode padrão UTF-8 e sugestões que vão surgindo ao longo da nossa interação com a ferramenta.



## Visual Studio Code 

Desenvolvimento pela Microsoft, é um editor de texto muito simples e interativo que conta com inúmeras extensões, temas variados e integração com tecnologias como NodeJS e Java.

Comparando-se em performance, o VS Code demonstra um grande desempenho de inicialização, codificação e execução dos programas desenvolvidos. 



**Resumindo:**

- **Eclipse:** Mais utilizada no ambiente corporativo com interface bem interativa.
- **NetBeans:** Utilizada mais em ambiente acadêmico e com versão de menu e assistentes em português.
- **IntelliJ:** Ofereça muitos recursos de inteligência para agilidade de nosso desenvolvimento com sugestões de ações bem assertivas. 
- **VS Code:** Hoje muito utilizada por desenvolvedores que já atuam com NodeJs ou plataformas de front-end.

### Maven.

É uma ferramenta de autmação de compilação utilizada primariamente em projetos Java, mas hoje também é utilizada para construir e gerenciar projetos escritos em C#, Ruby, Scala e outras linguagens. 



## Atalhos

Todas as IDE's possuem combinações de comandos que são facilitadores para criação de arquivos, digitação de códigos, localização e execução das classes em nosso programa, além de comando já pré-definidos de linhas de código (Code Snippet).

**Code Snippets**, que numa tradução livre seria algo como "retalhos de código", são pedaços de código pré-fabricados e parametrizáveis que podemos inserir em nossa aplicação. 



Comando

- Criar o método princpal main.

**Eclipse -** main, ctrl + espaço, enter

**IntelliJ -** psvm

- Localizar recurso/arquivo.

**Eclipse -** ctrl + shift + R

**IntelliJ -** ctrl + shift + N

- Método de Impressão no Console.

**Eclipse -** sysout, ctrl + espaço

**IntelliJ -** sout

- Renomeando variável ou método.

**Eclipse -** alt + shift + R

**IntelliJ -** shift + F6

- Localizar a utilização de um método.

**Eclipse -** ctrl + shift + G

**IntelliJ -** alt + F7

- Entrar na implementação do método.

**Eclipse -** ctrl + click

**IntelliJ -** ctrl + click
