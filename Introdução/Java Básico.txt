# Java Básico 

O Java é composta por um conjunto de funcionalidades (features) elas são super recomendadas para um entusiasta em programação conhecer o máximo que puder.

### Simples 

##### *Se mantivermos uma consistência na aprendizagem sobre Java, ela se torna simples.*

Java é uma linguagem de programação simples e fácil de entender, pois não contém complexidades que existiam em linguagens de programação anteriores. Na verdade, a simplicidade era o objetivo do design dos povos **Javasoft**, porque ele tem que funcionar em dispositivos eletrônicos onde menos memória/recursos estão disponíveis.



## Orientada a Objeto

##### *Tudo na linguagem Java é orientada a objetos, com exceção dos tipos primitivos.*

Java é uma linguagem de programação Orientada a Objetos. Isso significa que em Java tudo é escrito em termos de CLASSES e OBJETOS.

Para começar a desenvolver softwares de forma consistente com Java, é necessário compreender os pilares da **Programação Orientada a Objetos (POO). **

Eles são:

1. Classe e Objeto. 
2. Encapsulamento.
3. Abstração. 
4. Herança.
5. Polimorfismo.



### Plataforma independente 

O objetivo de design do javasoft people é desenvolver uma linguagem que funcione em qualquer plataforma. Uma plataforma é o ambiente de hardware ou software no qual um programa é executado.

Em Java, o código-fonte Java é compilado para bytecode e esse bytecode não está vinculado a nenhuma plataforma.

Ex: Windows, Linux, em um Micro-ondas ou uma aplicação mobile. 

***Dentro da plataforma java tem essa disponibilidade de rodar essa aplicação.***

Porque a ideia do time é criar uma extensão, ou um tipo no qual consiga ser interpretado em qualquer uma dessas plataformas.

E esse tipo é denominado de **BYTECODE.** O bytecode é livre de qualquer plataforma.



### Portátil 

O conceito WORA (Write Once Run Anywhere) e o recurso independente de plataforma tornam o Java portátil. Agora, usando a linguagem de programação Java, os desenvolvedores podem obter o mesmo resultado em qualquer máquina, escrevendo o código apenas uma vez. A razão por trás disso é JVM e bytecode. 

 

### Robusta 

A linguagem de programação Java é robusta, o que significa que é capaz de lidar com encerramento inesperado de um programa.

- Ele usa um gerenciamento de memória forte.
- Java fornece uma coleta de lixo automática.
- Há um tratamento de exceção e mecanismo e verificação de tipo em Java.



### Segura 

Problemas como ameaças de vírus, adulteração, espionagem ou representação podem ser tratados ou minimizados usando Java. E nossa construção de aplicativo usando Java também precisa de algum tipo de segurança. Por isso, a linguagem também fornece recursos de segurança para os programadores. Também existem **Recursos de criptografia e decriptografia** para proteger seus dados contra espionagem e adulteração na internet.



### Interpretada 

Nas linguagens de programação, aprendemos que eles usam o compilador ou o interpretador, mas a linguagem de programação Java usa os dois. Os programas Java são compilados para gerar arquivos de bytecode e a JVM ( Java Virtual Machine) interpreta o arquivo de bytecode durante a execução. 

**Ex.**

Os programas são compilados para gerar o arquivo bytecode.

- ***O que é a compilação?*** 

É tirar um arquivo com uma determinada instrução, estrutura ou extensão, e compilar/converter para uma estrutura que vai ser interpretada pelo nosso Hardware(Pelo sistema operacional que está sendo executado) ou pela JVM (Java Virtual Machine) que é a estrutura da linguagem que é capaz de ler o bytecode.

E por fim essa é a proposta de compilação.

- ***E o que a JVM determina?***

Que está interpretando, entendendo todo o conteúdo que o bytecode contém, dentro do processo que foi executado anteriormente da compilação.



**A linguagem Java tem a capacidade de haver etapa de compilaão e a JVM é capaz de interpretar todo o conteúdo existente do bytecode.**



## Multi-thread 

Thread é um subprocesso leve e independente de um programa em execução (ou seja, processo) que compartilha recursos. Multi-threading é o nome dado ao processo de vários threads sendo executados simultaneamente. 

***Pelo fato do Java der Multi-thread ele consegue nos proporcionar uma alta performance de execução.***
